# Infrastructure as Code (IaC) Portfolio

This repository is a collection of Infrastructure as Code (IaC) projects, showcasing hands-on experience with Terraform, Ansible, CloudFormation, and other automation tools for managing cloud infrastructure.

Each subfolder represents a separate project with its own setup, configuration, and documentation.

---

## Project Structure

- `terraform-aws-project/` - AWS environment deployed with Terraform (VPC, subnet, EC2, security groups, Docker installation, etc.)  
- `ansible-project/` - Example Ansible playbooks and roles for server provisioning and configuration  
- `cloudformation-project/` - CloudFormation templates for AWS resource deployment  

Each project folder contains a `README.md` with detailed documentation, instructions, and screenshots.

---

## How to Explore

1. Navigate into any project folder to view the detailed documentation.  
2. Follow the individual project instructions to initialize, deploy, or test the IaC code.  
3. Screenshots and example outputs are included in each project folder to demonstrate functionality.

---

## Notes

- Sensitive information (e.g., private keys, secrets) has been excluded from this repository.  
- Each project contains `.gitignore` files to ensure safe version control practices.  

---

## Goals of This Repository

- Demonstrate practical skills in cloud infrastructure automation  
- Provide a portfolio of IaC projects for career development  
- Share structured, reusable templates and best practices for deploying infrastructure
